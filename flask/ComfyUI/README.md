🚀 프로젝트 개요: ComfyUI 기반 AI 이미지 생성 웹 서비스이 프로젝트는 노드 기반의 ComfyUI(Stable Diffusion) 엔진을 백엔드로 활용하고, Flask를 통해 사용자 맞춤형 웹 인터페이스를 제공하는 AI 이미지 생성 서비스입니다. 사용자가 입력한 텍스트 프롬프트를 AI 모델이 해석하여 이미지를 생성하고, 이를 실시간으로 웹에서 확인할 수 있도록 설계되었습니다.🛠 기술 스택 (Tech Stack)구분기술 스택주요 역할FrontendBootstrap 5, JS, CSS3반응형 레이아웃 구현 및 테마(Dark/Light) 관리 BackendPython, FlaskAPI 오케스트레이션, 워크플로 관리 및 이미지 데이터 처리 AI EngineComfyUI, SD v1.5노드 기반 이미지 생성 워크플로 실행 및 확산 모델(Diffusion Model) 구동 Infra/DevOpsDocker, PM2NVIDIA GPU 가속 기반 컨테이너 환경 구축 및 서비스 무중단 운영 🔗 기술 스택 간 연결 및 아키텍처전체 시스템은 **'사용자 요청 - 서버 중계 - AI 생성 - 결과 반환'**의 4단계 프로세스로 연결됩니다.Frontend → Backend (Flask):사용자가 웹 폼에 긍정/부정 프롬프트를 입력하고 제출하면, Flask 서버는 이를 POST 요청으로 수신합니다.Backend (Flask) → AI Engine (ComfyUI):서버는 사전에 정의된 JSON 워크플로 파일을 로드합니다.random 라이브러리로 생성한 시드(Seed)값과 사용자 프롬프트를 로드된 JSON의 특정 노드 ID(노드 3, 6, 7번 등)에 주입하여 동적으로 워크플로를 업데이트합니다.수정된 JSON 데이터를 ComfyUI의 /prompt 엔드포인트로 전송하여 생성 큐에 등록합니다.API Polling & Monitoring:이미지 생성은 비동기적으로 이루어지므로, Flask 서버는 /history/{prompt_id} API를 주기적으로 호출(Polling)하여 생성 완료 상태를 추적합니다.Result Retrieval:생성이 완료되면 히스토리 데이터에서 이미지 파일명을 추출하고, ComfyUI의 /view API를 통해 이미지 파일 경로를 구성하여 프런트엔드에 전달합니다.⚙️ 주요 설정 및 핵심 구현 사항1. 동적 워크플로 제어 (JSON Manipulation)핵심: 고정된 UI를 넘어 API 형태의 확산 모델 제어.설정: ComfyUI에서 추출한 API용 JSON을 파이썬 딕셔너리 형태로 핸들링하여, 코드 레벨에서 모델의 파라미터(시드, 샘플링 단계, CFG 스케일 등)를 자유롭게 변경할 수 있도록 구현했습니다.2. GPU 가속 컨테이너 환경설정: docker run --gpus=all 옵션을 사용하여 컨테이너 내부에서도 호스트의 NVIDIA GPU 자원을 활용할 수 있도록 설정했습니다.효과: 복잡한 딥러닝 종속성 문제를 해결하고, 어떤 환경에서도 동일하게 작동하는 AI 인프라를 구축했습니다.3. 사용자 경험 최적화 (UX)테마 시스템: localStorage를 활용하여 브라우저 수준에서 사용자의 다크/라이트 모드 설정을 유지하게 하였고, CSS 변수(--bg-color 등)를 통해 일관된 톤앤매너를 유지했습니다.자산 관리(Assets): /history API 전체 응답을 파싱하여, 단순 일회성 생성이 아닌 과거 생성 이력을 리스트 형태로 관리하는 기능을 추가했습니다.💡 포트폴리오 한 줄 평:"노드 기반 AI 도구를 단순 활용하는 것을 넘어, REST API와 Flask 백엔드를 연동하여 실서비스 가능한 수준의 AI 이미지 렌더링 파이프라인을 구축했습니다."
